## 系统要求

### 最低配置（CPU模式）
- **操作系统**: Windows 10/11 或 Linux (Ubuntu 18.04+)
- **Python**: 3.9 或更高版本
- **内存**: 至少 8GB RAM
- **磁盘空间**: 至少 10GB 可用空间
- **处理器**: 支持AVX指令集的CPU

### 推荐配置（GPU模式）
- **操作系统**: Windows 10/11 或 Linux (Ubuntu 18.04+)
- **Python**: 3.9 或更高版本
- **内存**: 至少 16GB RAM
- **磁盘空间**: 至少 20GB 可用空间
- **处理器**: 支持AVX指令集的CPU
- **GPU**: NVIDIA GPU（支持CUDA 11.7或更高版本）
  - 推荐显存: 8GB 或更多
- **CUDA**: 11.7 或更高版本(作者的显卡是RTX5060,cuda版本为12.8)
- **cuDNN**: 8.9 或更高版本

## 快速开始

如果您已经配置好 Python 环境，可以按照以下快速步骤开始：

### 方式一：使用 Conda（推荐）

```bash
# 1. 克隆或下载项目到本地
cd d:\Projects\Grounding-Dino-Training

# 2. 创建并激活 Conda 环境
conda env create -f GroundingDINO/environment.yaml
conda activate dino

# 3. 安装额外的 pip 依赖
cd GroundingDINO
pip install -r requirements.txt

# 4. 启动 UI
cd ..
.\launch_ui.ps1
```

### 方式二：使用 venv

```bash
# 1. 克隆或下载项目到本地
cd d:\Projects\Grounding-Dino-Training

# 2. 创建虚拟环境
python -m venv venv_groundingdino

# 3. 激活虚拟环境
# Windows (PowerShell):
& .\venv_groundingdino\Scripts\Activate.ps1
# Windows (CMD):
venv_groundingdino\Scripts\activate.bat
# Linux/Mac:
source venv_groundingdino/bin/activate

# 4. 安装依赖
cd GroundingDINO
pip install -r requirements.txt

# 5. 启动 UI
cd ..
.\launch_ui.ps1
```

## 详细配置步骤

### 步骤 1: 安装 Python

#### Windows 系统

1. 访问 [Python 官网](https://www.python.org/downloads/)
2. 下载 Python 3.9 或更高版本的安装程序
3. 运行安装程序，**务必勾选 "Add Python to PATH"**
4. 验证安装：
   ```powershell
   python --version
   pip --version
   ```

#### Linux 系统

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install python3.9 python3.9-venv python3-pip

# 验证安装
python3 --version
pip3 --version
```

### 步骤 2: 创建虚拟环境

#### 使用 Conda（推荐）

Conda 提供了更好的环境隔离和依赖管理，特别是对于 GPU 相关的包。

```bash
# 1. 安装 Miniconda（如果尚未安装）
# 下载地址: https://docs.conda.io/en/latest/miniconda.html

# 2. 创建环境
cd d:\Projects\Grounding-Dino-Training
conda env create -f GroundingDINO/environment.yaml

# 3. 激活环境
conda activate dino
```

#### 使用 venv

如果您不想使用 Conda，可以使用 Python 内置的 venv：

```bash
# 1. 创建虚拟环境
cd d:\Projects\Grounding-Dino-Training
python -m venv venv_groundingdino

# 2. 激活虚拟环境
# Windows (PowerShell):
& .\venv_groundingdino\Scripts\Activate.ps1

# Windows (CMD):
venv_groundingdino\Scripts\activate.bat

# Linux/Mac:
source venv_groundingdino/bin/activate
```

**注意**: 如果在 Windows PowerShell 中遇到执行策略错误，运行以下命令：

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 步骤 3: 安装项目依赖

确保虚拟环境已激活，然后安装依赖：

```bash
# 进入 GroundingDINO 目录
cd GroundingDINO

# 安装所有依赖
pip install -r requirements.txt
```

**依赖包说明**:
- `torch`: PyTorch 深度学习框架
- `torchvision`: PyTorch 计算机视觉库
- `transformers`: Hugging Face Transformers 库（用于 BERT 模型）
- `addict`: 字典增强工具
- `yapf`: Python 代码格式化工具
- `timm`: PyTorch 图像模型库
- `numpy`: 数值计算库
- `opencv-python`: OpenCV 计算机视觉库
- `supervision>=0.22.0`: 计算机视觉工具库
- `pycocotools`: COCO 数据集工具

### 步骤 4: 下载预训练模型

项目需要预训练的 Grounding DINO 模型权重文件：

```bash
# 创建 weights 目录（如果不存在）
mkdir -p GroundingDINO/weights

# 下载预训练模型
# 方式一：使用 wget（Linux/Mac）或 curl（Windows）
wget -O GroundingDINO/weights/groundingdino_swint_ogc.pth https://github.com/IDEA-Research/GroundingDINO/releases/download/v0.1.0-alpha/groundingdino_swint_ogc.pth

# 方式二：手动下载
# 访问: https://github.com/IDEA-Research/GroundingDINO/releases
# 下载 groundingdino_swint_ogc.pth 文件
# 放置到 GroundingDINO/weights/ 目录下
```

### 步骤 5: 配置 BERT 模型缓存

项目使用 BERT 模型进行文本编码。首次运行时会自动从 Hugging Face Hub 下载。

**本地缓存说明**:
- 项目已包含本地缓存的 BERT 模型
- 缓存路径: `huggingface_cache\transformers\models--bert-base-uncased`
- 系统会自动从本地缓存加载，无需重新下载

如果需要重新下载或更新模型，可以设置环境变量：

```bash
# Windows (PowerShell)
$env:HF_HOME = "d:\Projects\Grounding-Dino-Training\huggingface_cache"

# Linux/Mac
export HF_HOME="/path/to/your/huggingface_cache"
```

## GPU模式配置

如果您有 NVIDIA GPU，可以配置 GPU 模式以获得更快的推理速度。

### 步骤 1: 安装 CUDA Toolkit

1. 访问 [NVIDIA CUDA Toolkit 下载页面](https://developer.nvidia.com/cuda-downloads)
2. 选择您的操作系统和版本
3. 下载并安装 CUDA Toolkit 11.7 或更高版本
4. 验证安装：
   ```bash
   nvcc --version
   ```

### 步骤 2: 安装 cuDNN

1. 访问 [NVIDIA cuDNN 下载页面](https://developer.nvidia.com/cudnn)（需要注册 NVIDIA 账号）
2. 下载与 CUDA 版本匹配的 cuDNN（建议 8.9 或更高版本）
3. 解压并复制文件到 CUDA 安装目录：
   ```bash
   # Windows
   # 将解压后的文件复制到 C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.x\

   # Linux
   sudo cp cuda/include/cudnn*.h /usr/local/cuda/include
   sudo cp -P cuda/lib64/libcudnn* /usr/local/cuda/lib64
   sudo chmod a+r /usr/local/cuda/include/cudnn*.h /usr/local/cuda/lib64/libcudnn*
   ```

### 步骤 3: 安装 GPU 版本的 PyTorch

```bash
# 激活虚拟环境
conda activate dino  # 或激活您的 venv 环境

# 卸载 CPU 版本的 PyTorch（如果已安装）
pip uninstall torch torchvision torchaudio

# 安装 GPU 版本的 PyTorch
# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# 或使用 Conda 安装
conda install pytorch torchvision torchaudio pytorch-cuda=11.8 -c pytorch -c nvidia
```

### 步骤 4: 验证 GPU 安装

```bash
# 启动 Python
python

# 在 Python 中运行以下代码
import torch
print(f"PyTorch version: {torch.__version__}")
print(f"CUDA available: {torch.cuda.is_available()}")
if torch.cuda.is_available():
    print(f"CUDA version: {torch.version.cuda}")
    print(f"GPU count: {torch.cuda.device_count()}")
    print(f"GPU name: {torch.cuda.get_device_name(0)}")
```

预期输出应该显示 `CUDA available: True`。

## 常见问题解决

### 问题 1: ModuleNotFoundError: No module named 'cv2'

**原因**: OpenCV 未正确安装

**解决方案**:
```bash
pip install opencv-python==4.8.1.78
```

### 问题 2: CUDA out of memory

**原因**: GPU 显存不足

**解决方案**:
- 减小批处理大小（batch size）
- 使用较小的图像分辨率
- 关闭其他占用 GPU 显存的程序

### 问题 3: ImportError: DLL load failed

**原因**: 缺少必要的系统库或 CUDA/cuDNN 配置不正确

**解决方案**:
- 确保已正确安装 CUDA Toolkit 和 cuDNN
- 将 CUDA bin 目录添加到系统 PATH：
  ```bash
  # Windows
  # 添加到 PATH: C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v11.x\bin
  ```
- 重新安装 PyTorch：
  ```bash
  pip uninstall torch torchvision torchaudio
  pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
  ```

### 问题 4: 网络连接问题（无法下载模型）

**原因**: 网络限制或防火墙阻止下载

**解决方案**:
- 使用代理：
  ```bash
  pip install --proxy http://user:password@proxyserver:port -r requirements.txt
  ```
- 手动下载模型文件并放置到正确位置
- 使用国内镜像源：
  ```bash
  pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
  ```

### 问题 5: PowerShell 执行策略错误

**错误信息**: 无法加载文件，因为在此系统上禁止运行脚本

**解决方案**:
```powershell
# 临时允许执行脚本
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 或临时绕过执行策略
powershell -ExecutionPolicy Bypass -File launch_ui.ps1
```

### 问题 6: 虚拟环境激活失败

**原因**: PowerShell 执行策略限制或路径问题

**解决方案**:
```powershell
# 方式一：修改执行策略
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 方式二：使用完整路径激活
& "d:\Projects\Grounding-Dino-Training\venv_groundingdino\Scripts\Activate.ps1"

# 方式三：使用 CMD 代替 PowerShell
# 在 CMD 中运行
venv_groundingdino\Scripts\activate.bat
```

### 问题 7: BERT 模型下载失败

**原因**: 网络问题或 Hugging Face Hub 访问受限

**解决方案**:
- 使用本地缓存（项目已包含）
- 设置离线模式：
  ```bash
  export TRANSFORMERS_OFFLINE=1
  export HF_DATASETS_OFFLINE=1
  ```
- 手动下载并放置到缓存目录
