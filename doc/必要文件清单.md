## 文件大小估算

### 必要文件（不含模型权重）
- 核心代码和配置：约 50-100 MB
- 文档文件：约 1-2 MB
- **总计：约 50-100 MB**

### 包含预训练模型权重
- 核心代码和配置：约 50-100 MB
- 预训练模型权重：约 1.7 GB
- **总计：约 1.8-2 GB**

### 包含Hugging Face缓存
- 核心代码和配置：约 50-100 MB
- Hugging Face缓存：约 400-500 MB
- **总计：约 450-600 MB**

### 完整包（包含所有）
- 核心代码和配置：约 50-100 MB
- 预训练模型权重：约 1.7 GB
- Hugging Face缓存：约 400-500 MB
- **总计：约 2.1-2.3 GB**

## 打包建议

### 方案一：最小包（推荐）
只包含核心代码和配置，新成员自行下载模型：
- 文件大小：约 50-100 MB
- 优点：体积小，传输快
- 缺点：新成员需要下载模型（约1.7GB + 400MB）

### 方案二：标准包
包含核心代码、配置和Hugging Face缓存：
- 文件大小：约 450-600 MB
- 优点：新成员无需下载BERT模型
- 缺点：需要下载预训练模型权重（约1.7GB）

### 方案三：完整包
包含所有文件（包括预训练模型权重）：
- 文件大小：约 2.1-2.3 GB
- 优点：开箱即用，无需下载任何模型
- 缺点：体积大，传输慢

## 使用打包脚本

项目提供了打包脚本 `doc/create_package.ps1`，可以自动打包必要文件：

```powershell
# 创建最小包
.\doc\create_package.ps1 -PackageType minimal

# 创建标准包（包含Hugging Face缓存）
.\doc\create_package.ps1 -PackageType standard

# 创建完整包（包含所有模型）
.\doc\create_package.ps1 -PackageType full
```

## 新成员使用指南

新成员收到压缩包后：

1. 解压压缩包
2. 阅读 `环境配置指南.md` 配置环境
3. 阅读 `README_启动说明.md` 启动项目
4. 根据需要阅读其他文档

## 注意事项

1. **模型权重文件较大**：如果选择不包含预训练模型权重，需要告知新成员下载地址
2. **Hugging Face缓存**：如果包含缓存，确保新成员的目录结构与发送时一致
3. **路径问题**：脚本中的绝对路径（如 `d:\Projects\Grounding-Dino-Training`）可能需要根据新成员的实际路径进行调整
4. **虚拟环境**：新成员需要根据 `环境配置指南.md` 创建自己的虚拟环境
