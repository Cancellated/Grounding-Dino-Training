# Grounding DINO 训练步骤指南

## 1. 环境准备

### 步骤 1: 激活虚拟环境

打开 PowerShell 并执行以下命令激活 Grounding DINO 的虚拟环境：

```powershell
cd d:\Projects\Grounding-Dino-Training
& .\venv_groundingdino\Scripts\Activate.ps1
```

### 步骤 2: 安装依赖

确保所有训练所需的依赖都已正确安装：

```powershell
cd GroundingDINO
pip install -r requirements.txt
```

## 2. 数据集准备

### 步骤 1: 准备图像数据

将您的训练图像放入一个文件夹中，例如 `d:\Projects\Grounding-Dino-Training\Images`

### 步骤 2: 生成 COCO 格式数据集

使用项目提供的工具自动生成 COCO 格式的标注文件：

```powershell
cd GroundingDINO\suite\demo
python create_coco_dataset.py \
  --image-directory "d:\Projects\Grounding-Dino-Training\Images" \
  --text-prompt "gypsum" \
  --box-threshold 0.15 \
  --text-threshold 0.10 `
    --export-dataset `
    --export-path "../custom_coco_dataset" `
    --annotated-images-path "../custom_annotated_images" `
    --weights-path "../../weights/groundingdino_swint_ogc.pth" `
    --config-path "../../groundingdino/config/GroundingDINO_SwinT_OGC.py"
```

**参数说明：**
- `--image-directory`: 训练图像所在文件夹
- `--text-prompt`: 要检测的目标类别（这里是 "gypsum"）
- `--box-threshold`: 边界框置信度阈值
- `--text-threshold`: 文本置信度阈值
- `--export-dataset`: 导出为 COCO 格式
- `--export-path`: 自定义 COCO 数据集导出路径（可选，默认：`coco_dataset`）
- `--annotated-images-path`: 自定义带标注图像导出路径（可选，默认：`images_with_bounding_boxes`）
- `--weights-path`: 预训练模型权重路径
- `--config-path`: 模型配置文件路径

### 步骤 3: 检查生成的数据集

如果使用默认设置，工具将在 `suite/demo` 文件夹下生成以下目录：
- `coco_dataset`: COCO格式数据集，包含 `labels.json`（标注文件）和 `data`（图像文件夹）
- `images_with_bounding_boxes`: 带边界框标注的图像

如果使用了自定义路径（`--export-path` 和 `--annotated-images-path`），则数据集和标注图像将保存在指定的路径中。

## 3. 模型训练

### 步骤 1: 进入训练目录

```powershell
cd d:\Projects\Grounding-Dino-Training\GroundingDINO
```

### 步骤 2: 运行训练脚本（单GPU）

```powershell
python suite/train_grounding_dino.py \
  --config-file groundingdino/config/GroundingDINO_SwinT_OGC.py \
  --data-dir "suite/demo/coco_dataset" \
  --batch-size 4 \
  --epochs 10 \
  --output-dir "./output/train_gypsum"
```

### 步骤 3: 多GPU训练（可选）

如果您有多个GPU，可以使用分布式训练加速：

```powershell
python -m torch.distributed.launch --nproc_per_node=2 suite/train_grounding_dino.py \
  --config-file groundingdino/config/GroundingDINO_SwinT_OGC.py \
  --data-dir "suite/demo/coco_dataset" \
  --batch-size 2 \
  --epochs 10 \
  --output-dir "./output/train_gypsum" \
  --distributed
```

**参数说明：**
- `--nproc_per_node`: GPU数量
- `--batch-size`: 每个GPU的批大小
- `--config-file`: 模型配置文件路径
- `--checkpoint-path`: 训练好的模型权重路径
- `--anno-path`: COCO格式标注文件路径
- `--image-dir`: 评估图像所在文件夹

## 4. 模型评估

### 步骤 1: 运行评估脚本

训练完成后，使用以下命令评估模型性能：

```powershell
cd suite/demo
python test_ap_on_coco.py \
  --config-file "../../groundingdino/config/GroundingDINO_SwinT_OGC.py" \
  --checkpoint-path "../../output/train_gypsum/checkpoint_best.pth" \
  --anno-path "./coco_dataset/annotations/instances_default.json" \
  --image-dir "./coco_dataset/images"
```

**评估指标说明：**
- AP: 平均精度 @ IoU=0.50:0.95
- AP50: 平均精度 @ IoU=0.50
- AP75: 平均精度 @ IoU=0.75
- AR: 平均召回率

## 6. 常见问题解决

### 问题 1: CUDA 内存不足

**解决方案：**
- 减小 `--batch-size` 参数
- 使用较小的图像分辨率

### 问题 2: 训练速度慢

**解决方案：**
- 确保使用GPU训练
- 增加 `--num-workers` 参数

### 问题 3: 模型不收敛

**解决方案：**
- 检查数据集标注质量
- 调整学习率（减小 `--lr` 参数）
- 增加训练轮数
